# GRID — Web App

An elegant, animated multi-page front-end for the GRID parking-congestion engine.
Built with **Vite + React + TypeScript + Tailwind + Framer Motion + Leaflet + Recharts**.

It reads the precomputed data produced by the Python pipeline (`../src/pipeline.py`)
from `public/data/grid.json` — the front-end does no heavy analytics itself, it just
renders and aggregates.

## Pages

| Route | What it does |
|---|---|
| `/` | Animated landing — live "city-map" hero backdrop seeded from real hotspots, word-reveal headline, count-up stats. |
| `/dashboard` | Leaflet map with independent **layer checkboxes** (heatmap / hotspot zones — uncheck both for a clean map), and the **Area Explorer**: click anywhere to drop a pin and get **exact** analytics for that radius (violations, impact, hour pattern, violation/vehicle mix, hotspot zones inside, **and a patrol forecast for the area**). Plus a ranked enforcement worklist. |
| `/forecast` | Patrol-forecast roster — risk-scored zones, trend badges, Movers, validation banner, risk map. |
| `/insights` | Animated Recharts: impact concentration, hour pattern, impact by violation/vehicle/division. |

### The pin-drop is exact, not sampled
Every violation lives in exactly one ~110 m grid cell, and the cells partition the
whole dataset (their counts sum to the city total). So aggregating the cells whose
centroid falls inside the pin radius gives **true totals** — verified in the build.

## Run locally

```bash
cd web
npm install
npm run dev        # → http://localhost:5173
```

The map base tiles need internet (CartoDB); the app + data are local. Leaflet's CSS
and JS are bundled (no runtime CDN).

## Build

```bash
npm run build      # type-checks + builds to dist/
npm run preview    # serve the production build at :4173
```

Routes are code-split (React.lazy), so the landing page stays light and Leaflet /
Recharts load only on the pages that use them.

## Deploy to Vercel

The repo includes `vercel.json`. Either:

- **Dashboard:** import the repo in Vercel, set **Root Directory = `web`**, framework
  auto-detects as Vite. Done.
- **CLI:** `cd web && npx vercel --prod`.

`public/data/grid.json` is committed, so the deployed site is fully static — no
backend needed.

## Refreshing the data

Re-run the Python pipeline from the repo root; it writes `web/public/data/grid.json`
automatically:

```bash
.venv/bin/python src/pipeline.py "jan to may police violation_anonymized791b166.csv"
```

Shareable deep-link for a pinned area: `/#/dashboard?pin=<lat>,<lon>,<radius>`.
